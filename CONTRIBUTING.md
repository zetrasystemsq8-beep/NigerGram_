# Contributing to NigerGram

Thanks for your interest in contributing! We welcome bug reports, improvements, and new features. Please follow these guidelines to make the process smooth for everyone.

1. Fork the repository
2. Create a feature branch from main:

```bash
git checkout -b feat/short-description
```

3. Make small, focused commits with clear messages. Follow Conventional Commits if possible, e.g. `fix(...)`, `feat(...)`, `chore(...)`.

4. Run tests and linters locally (if available):

```bash
flutter pub get
flutter analyze
```

5. Open a Pull Request against `main` with:
- A clear title
- A description of what you changed and why
- Steps to reproduce or test

6. Address review comments and keep PRs small and focused.

Code style
- Follow Dart/Flutter formatting: `dart format .`
- Keep functions small and files organized by feature under `lib/features/`

Thank you for contributing — we appreciate your help!
